## Requirements

### 1. Data Model
All setup cards are rendered from a `SETUPS` JavaScript array at the top of the `<script>` section. No hardcoded card HTML.

Each setup object has: `symbol, name, sector, direction, price, change_pct, status, bias, sma20, sma50, weekly_sma20, rsi, pct_from_high, above_sma20, above_sma50, above_weekly20, entry_zone, stop_loss, target_1, target_2, analysis, source, signals[]`

For RETEST status, add: `retest: { level, level_price, distance_pct, holding, healthy_volume, confidence, is_confluence, confluence_levels[] }`

### 2. Filter Bar (replaces old status-tabs)

Layout:
```
[All 13] [🔥 Hot 5] [🟢 Active 3] [⚠️ Alerts 5] [👁 Watch 3]     ← status tabs
🔍 [Search...]  [BULL] [MIXED] [BEAR]  Sort: [Priority ▾]           ← controls row
[All] [Tech] [Healthcare] [Energy] [Utilities] [Consumer] [Crypto]  ← sector chips
```

Behavior:
- Status tabs: mutually exclusive, filter by status group
- Bias toggles: multi-select, all ON by default
- Sector chips: multi-select, "All" deselects others
- Search: instant filter on keyup (debounced 150ms), matches symbol or name
- Sort dropdown: Priority (default), % Change, RSI, Alphabetical
- All badges show live counts

### 3. Card Rendering

Cards rendered by `renderCard(setup)` function. Different layouts per status:

- **RETEST**: Full expanded card with retest details (which level, holding?, volume health, confidence badge). Orange-red border, pulse animation.
- **TRIGGERED**: Green border, "NEW" badge, entry/stop/target levels
- **APPROACHING**: Orange border, distance to entry, gentle pulse
- **ACTIVE/TRAILING**: Green/blue border, range bar, SMA pills
- **TARGET_HIT**: Gold border, celebration
- **EXIT_***: Red border, exit warning banner
- **WATCHING**: Gray border, compact single-line-ish
- **DORMANT**: Ultra-compact, dimmed
- **STOPPED**: Red border, 60% opacity

Group headers auto-generated based on sort mode. When sorting by Priority, group by status category.
